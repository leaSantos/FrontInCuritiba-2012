Pra começo de conversa

Contexto

Ao longo da última década, os dispositos móveis sofreram inúmeras modificações, de celulares a smartphones, de teclado convencional a touch screen e teclados QWERTY. Esta evolução modificou a forma com que as pessoas interagem com estes dispositivos.

35% da população que possui celular, tem acesso à internet no dispositivo

O paradigma "mobile"

Desktop vs. Mobile

  • Interfaces de desktop e mobile
    • Mais homogêneas entre si, se assemelhando cada vez mais
  • Interação de desktop e mobile
    • Distintas
As interfaces criadas para cada dispositivo nos ajudam a distinguir os ambientes e também a forma com que o usuário interage

(Eis, 2011)

Fonte: A User Centered Approach to Mobile Design

Smartphones e interfaces touch

  • Dispositivos como os smartphones, estão em crescente substituição aos dispositivos convencionais
  • Popularização de smartphones com tela sensível ao toque
Criar interfaces touch é algo relativamente novo. Nós trouxemos ideias da interação com mouse para os dispositivos touch, mas grande parte das interações precisaram ser reinventadas. [...] Lembre agora na forma de como você gira uma imagem em um dispositivo touch e como você gira essa mesma imagem utilizando um mouse. A interface muda, o seu comportamento muda.

(Eis, 2011)

Melhores práticas de design e front-end

Interface

  • Navegação e menus
  • Ícones
  • Tarefas a serem desempenhadas (Apple, 2011; Android, 2011)
    • Ao utilizar um dispositivo, os usuários começam a realizar tarefas uma após a outra, alheios ao comportamento subjacente
    • Experiência deve ser contínua, atividade após atividade, tarefa após tarefa

Usabilidade (Apple, 2011)

  • Foco em tarefas primárias
  • Eleve o conteúdo com o qual as pessoas se importam
    • Liste as funcionalidades que os usuários possam gostar
    • Determine a audiência
  • Dar ao usuário um caminho lógico a seguir
  • Utilização simples e óbvia
  • Terminologia “centrada no usuário”

Experiência do Usuário

  • Qual a necessidade de seus usuários ao acessar o seu site através de um dispositivos móvel?
  • Que recursos ele utilizam?
  • Quais recursos são cruciais para eles quando estão acessando via mobile?
  • Quais as frustrações?
  • Quais dispositivos são utilizados para este acesso?

Fonte: A User Centered Approach to Mobile Design

Design Responsivo ou Versão Mobile?

  • Reiterando: experiências diferentes
  • Técnicas igualmente válidas, variando de acordo com o escopo do site
  • Sites de conteúdo
    • Reduzir algumas informações
    • Rearranjo de layout para beneficiar diferentes dispositivos
    • Utilizar os padrões de interface de mobile
  • Aplicações web
    • Subconjunto de funcionalidades da aplicação desktop
    • Tarefas mais focadas
    • Corte de alguns caminhos de interação

Front-end (Eis, 2011; Pagani, 2010)

Teclas de atalho para executar ações
Ações importantes expostas na interface
Controle da área de toque
Concisão
Conteúdo minimizado: ações, textos, links, imagens e controles de formulários que sejam mais relevantes
Minimizar erros em formulários
Labels acima do campo
Espaçamento adequado dos campos
Cuidado com a área de clique do botão submit (Envio involuntário dos dados)
Utilizar o input type adequado (HTML5!)
Minimizar input
Pré-seleção para valores em campos de formulário
Utilizar listas de seleção, radio buttons e outros controles que não requerem digitação
